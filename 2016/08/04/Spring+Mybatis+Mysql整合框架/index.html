Spring+Mybatis+Mysql整合框架

  • Spring、Mybatis、Oracle
JAVA

在现代的大型应用中，都要求有高并发、高可用性等的要求，而这些需求我们在WEB系统中常常借助于框架来帮助我们达到我们的要求。本文介绍的是博主真是工作中使用的框架整合，也就是Spring+Mybatis+Oracle的整合使用。

Spring整合Mybatis的优点

简单总结了以下几条Spring整合Mybatis使用的优点：

  • 在Mybatis编写的SQL中，可以使用别名的方式直接映射到对应实体类属性。
  • 在Spring配置文件中，可以配置自动扫描方式，从而Spring可以自动帮助我们扫面配置目录下的DAO。
  • 通过整合Mybatis后，Spring会帮助我们自动实现DAO接口，从而可以不需要手动的去编写DAO接口，并且可以自动注入到Spring容器中。但是此种方式也是有局限性的，特别是对于比价复杂的业务逻辑处理。
  • Spring整合Mybatis后，我们可以自由的编写SQL脚本，可以自己传入参数，并且传入参数后Mybatis会自动赋值。可以自定义SQL语句，就给予开发人员一个很大的好处，不必像Hibernate这种框架样必须使用一些语法，很大程度上给予开发人员便捷。

简单介绍下Spring IOC

Spring IOC（Inversion of Control）即控制反转，也被称为DI（依赖注入），IOC是一种设计思想，而不是技术。IOC意味着，创建的对象需要交到容器中控制，而不是传统的直接在对象中使用。理解IOC需要从以下几点进行思考：谁控制谁，控制了什么？为什么是反转（有反转应该也有正转）？哪些方面被反转了？

  • 谁控制了谁，控制了什么？传中面向对象编程中，我们是直接在对象内部通过new的方式直接创建对象，而IOC中是有一个IOC容器来管理这些对象的创建。也就是说IOC容器控制了这些对象的创建。控制的东西，就是外部资源的获取，这里资源不只是对象还有文件、数据源等。
  • 传统的编程中，我们是直接在对象中创建并获取依赖的对象，这个就是正转。而IOC容器中，对象是在容器中控制并创建的，在需要使用的对象中，对象是通过容器自动注入所需要依赖的对象，也就是被动的接受依赖的对象，这就是反转。反转的东西，就是对象的获取。

使用Spring IOC管理Bean的原因

基于以下几点优点，我们使用Spring IOC来管理Bean：

  • 对象统一管理
  • 规范Bean的生命周期
  • 灵活的依赖注入
  • 一致的获取对象

使用Spring IOC注入方式及使用场景

1、使用XML配置文件注入

使用XML配置文件注入，一般是需要引用第三方类库，比如dataSource等等。或者需要使用命名空间，比如AOP、Context、MVC等等。

2、使用Spring注解方式注入

使用Spring注解方式十分方便，我们可以直接在对象声明处，使用Spring的相关注解，直接把Bean注入到变量中。一般如果引用的是项目本身开发的类库，可以使用注解方式注入，简单方便，注解例举，比如: @Service、@Controller等等。

3、使用JAVA类的方式注入

这种情形一般是需要使用代码逻辑来控制对象的创建，比如需要自定义修改库类等等。

使用Spring管理事物方式介绍

1、使用ProxyFactoryBean+XML方式为早期Spring使用的事物管理方式。    
2、使用tx:advice+aop命名空间，一次配置永久有效，这种方式就方便了后面开发中使用。       
3、使用@Tranactional注解方式，在需要使用事物管理的地方定义事物管理方法。

Spring整合Mybatis实例

1、Spring配置文件Spring.xml 

<?xml version="1.0" encoding="UTF-8" ?>
<beans xmlns="http://www.springframework.org/schema/beans"
xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
xmlns:context="http://www.springframework.org/schema/context"
xsi:schemaLocation="http://www.springframework.org/schema/beans
http://www.springframework.org/schema/beans/spring-beans-3.0.xsd 
http://www.springframework.org/schema/context  
http://www.springframework.org/schema/context/spring-context-3.0.xsd">

<!-- 整合配置Mybatis -->
<!-- 1.配置数据库源 ，获取properties属性：${}-->
<context:property-placeholder location="classpath:jdbc.properties"/>

<!-- 2.配置数据源属性  -->
<bean id="dataSource" class="com.mchange.v2.c3p0.ComboPooledDataSource">
    <!-- 配置链接池的相关信息 -->
    <property name="driverClass" value="${jdbc.driver}" />
    <property name="jdbcUrl" value="${jdbc.url}" />
    <property name="user" value="${jdbc.username}" />
    <property name="password" value="${jdbc.password}"/>

    <!-- 配置c3p0相关私有属性 -->
    <property name="maxPoolSize" value="30" />
    <property name="minPoolSize" value="10" />
    <!-- 关闭连接后不自动commit -->
    <property name="autoCommitOnClose" value="false" />
    <!-- 设置数据库连接超时时间 -->
    <property name="checkoutTimeout" value="1000" />
    <!-- 设置自动获取数据库连接次数 -->
    <property name="acquireRetryAttempts" value="2" />
</bean>

<!-- 3.配置SqlSessionFactory信息 -->
<bean id="sqlSessionFactory" class="org.mybatis.spring.SqlSessionFactoryBean">
    <!-- 配置数据源引用 -->
    <property name="dataSource" ref="dataSource" />
    <!-- 配置Mybatis全局配置文件mybatis-config.xml路径 -->
    <property name="configLocation"  value="classpath:mybatis-config.xml" />
    <!-- 扫描Entity包，使用别名。 -->
    <property name="typeAliasesPackage" value="com.lanceng.entity" />
    <!-- 扫描Sql配置文件：Mapper需要的xml配置文件 -->
    <property name="mapperLocations" value="classpath:mapper/*.xml" />
</bean>

<!-- 4.配置扫描DAO接口包 ，动态实现DAO，自动注入到Spring容器。-->
<bean class="org.mybatis.spring.mapper.MapperScannerConfigurer">
    <!-- 注入SqlSessionFactory -->
    <property name="sqlSessionFactoryBeanName" value="sqlSessionFactory" />
    <!-- 需要扫描DAO接口包 -->
    <property name="basePackage" value="com.lanceng.dao" />
</bean>
</beans>

3、测试用例SeckillDaoTest.java 

package com.lanceng.dao;

import java.sql.Connection;
import java.sql.Date;
import java.sql.DriverManager;
import java.util.Iterator;
import java.util.List;
import java.util.Map;
import java.util.Properties;

import javax.annotation.Resource;

import org.junit.Test;
import org.junit.runner.RunWith;
import org.mybatis.spring.annotation.MapperScan;
import org.springframework.test.context.ContextConfiguration;
import org.springframework.test.context.junit4.SpringJUnit4ClassRunner;

import com.lanceng.entity.Seckill;

/**
  * @描述：配置Spring和Junit整合，Junit启动时加载SpringIOC容器。
  * @RunWith: Junit启动时加载SpringIOC容器
  * @ContextConfiguration: 告诉Junit，Spring的配置文件路径。
  * @author lanceng
  * @时间：2016年6月28日21:53:48
  */
@RunWith(SpringJUnit4ClassRunner.class)
@ContextConfiguration({ "classpath:spring/spring-dao.xml" })
public class SeckillDaoTest
{

//注入DAO实现类
@Resource
private SeckillDao seckillDao;

@Test
public void testQuerySeckillById()
{
    long seckillId = 1004;
    Seckill seckill = seckillDao.querySeckillById(seckillId);
    if (seckill != null)
    {
        System.out.println(seckill.getName());
        System.out.println(seckill);
    }
    else
    {
        System.out.println("seckill is null");
    }
}

@Test
public void testQueryPageSeckill()
{
    //Java不会保存形参的记录：queryPageSeckill(int offSet, int limit) -> queryPageSeckill(arg0, arg1)
    List<Seckill> seckills = seckillDao.queryPageSeckill(0, 100);
    for (Seckill seckill : seckills)
    {
        System.out.println(seckill);
    }
}

@Test
public void testReduceNumber()
{
    Date seckillDate = new Date(20160628);
    int updateAccounts = seckillDao.reduceNumber(1000L, seckillDate);
    System.out.println("updateAccounts=" + updateAccounts);
}

@Test
public void testGetConnection()
{
    String url = "jdbc:mysql://localhost:3306/seckill";
    String driver = "com.mysql.jdbc.Driver";
    String user = "root";
    String passwd = "123456";

    try
    {
        Class.forName(driver);
    }
    catch (Exception e)
    {
        System.out.println("Get Connection failed!!!");
    }

    try
    {
        Connection con = DriverManager.getConnection(url, user, passwd);

        System.out.println("Get Connection Success!!!");

        Properties properties = con.getClientInfo();
        Iterator<Map.Entry<Object, Object>> it = properties.entrySet().iterator();
        while (it.hasNext())
        {
            Map.Entry<Object, Object> entry = it.next();
            System.out.println(entry.getKey() + "---------------" + entry.getValue());
        }
    }
    catch (Exception e)
    {
        e.printStackTrace();
    }
}
}
