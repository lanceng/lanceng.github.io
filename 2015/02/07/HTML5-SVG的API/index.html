HTML5-SVG的API

  • svg api
HTML5

图形的两个概念：栅格图形和矢量图形。在栅格图形中，图像是有一组二维像素网格表示。HTML5的Canvas 2d API就是一款栅格图形API。通过Canvas API绘制图形，其实就是更新canvas的像素。PNG和JPEG是两种栅格图形的格式。

矢量图形则大相径庭。在矢量图形中，图像是有数学描述的几何形状表示。矢量图像包括使用高级几何图形（比如线和形状）绘制图像所需的全部信息。顾名思义，SVG就是矢量图形的一种。同HTML一样，SVG是一种格式，有自己的API。SVG同DOM API结合形成了一种矢量图形的API。尽管可以将PNG等栅格图形嵌入到SVG中，但从根本上讲，SVG是一种矢量格式。

SVG的创建

<svg height="100" height="100">
    <circle x="50" y="50" r="50" />
</svg>

SVG的缩放

缩放SVG图像不会导致其质量下降。SVG文档在呈现的时候会保留构成它的矢量信息。这与像素图形不同，放大canvas和图像这样的像素图形后，图像会变得模糊。其原因在于图像是由像素组成，且只能在更高分辨率下重新采样。基础信息–构成图像的路径和图形–在图像完成绘制后便丢失了。

变换SVG元素

SVG中有些组织元素，可以将多个元素结合起来，标签就是其中一个，代表“组”（group）。组内成员可通过ID来引用。此外，还可以将组作为一个整体进行变换。





SVG元素的复用

在SVG中，可以使用标签来定义留待将来使用的内容。然后在SVG中使用标签来连接到吃元素定义的内容。

<svg width="200" height="200">
    <defs>
        <g id="ShapeGroup">
            <rect x="10" y="10" width="100" height="80" stroke="red" fill="#ccc"/>
            <circle cx="120" cy="80" r="40" stroke="blue" fill="yellow" stroke-width="3"/>
        </g>
    </defs>

    <use xlink:href="#ShapeGroup" transform="translate(60, 0) scale(0.5)" />
    <use xlink:href="#ShapeGroup" transform="translate(120, 60) scale(0.4)" />
    <use xlink:href="#ShapeGoup" transform="translate(200, 120) scale(0.25)" />
</svg>

SVG的图案背景及渐变

图案引用及渐变事例： 

<svg width="200" height="200">
    <defs>
        <pattern id="GravelPattern" patternUnits="userSpaceOnUse" x="0" y="0" width="100" height="67" viewBox="0 0 100 67">
            <image x="0" y="0" width="100" height="67" xlink:href="test.jpg"></image>
        </pattern>
        <linearGradient id="RedBlackGradient">
            <stop offset="0%" stop-color="#000"></stop>
            <stop offset="100%" stop-color="#f00"></stop>
        </linearGradient>
    </defs>

    <rect x="10" y="20" width="100" height="80" stroke="red" fill="url(#RedBlackGradient)" stroke-width="5" />
    <circle cx="120" cy="80" r="40" stroke="#00f" fill="url(#GravelPattern)" stroke-width="8" />
</svg>

SVG路径

SVG不仅包含简单的图形，还包含自由形态的路径。path元素又一个d属性，d代表数据（data）。在d属性的值中，你能够制定一系列的路径绘制命令。每条命令都可能带有坐标参数。一些命令的含义为：M代表移至（moveTo），L代表划线至（lineTo），Q代表二次曲线，Z代表封闭合路径。如果你能想起canvas API，并非巧合。一个简单例子： 

<path d="M-25, -50 L-10, -80 L-20, -80 L-5, -110 L-15, -110 L0, =140 L15, -110 L5, -110 L20, -80 L10, -80 L25, -50 Z" id="Canopy"></path>

SVG文本

SVG也支持文本，在浏览器中，文本是可选择的。如果选用SVG文本，浏览器和搜索引擎能够支持用户搜索SVG text元素中的文本。这对可用性和可访问性有很大的益处。
SVG文本的属性类似于HTML里的CSS样式规则。包括一些font font-size font-family font-weight等等属性。 

<text x="10" y="50" font-size="40px" font-family="微软雅黑" font-weight="bold" fill="#0ff">
    Happy Trails!
</text>
