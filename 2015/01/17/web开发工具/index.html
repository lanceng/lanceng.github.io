Sublime Text brackets与webstorm对比

代码提示补全支持

  • Sublime Text ：Javascript、JQuery、HTML5 、安装插件可支持angular、bootstrap、Jasmine kama等前端框架、c#等多种语言。
  • brackets ：Javascript、JQuery 、html
  • Webstorm：avascript、JQuery、HTML5、安装插件可支持angular、bootstrap、Jasmine kama等前端框架。浏览器前缀支持。

    代码风格检查

  • Sublime Text ：安装jshint csslint插件可支持，可自定义风格。
  • brackets ：安装jshint csslint插件可支持，可自定义风格。
  • Webstorm：自身集成jshint csslint，可自定义风格。

Js调试支持程度

  • Sublime Text ：安装Web Inspector插件支持，支持添加多对象监视，支持实时预览(http://sokolovstas.github.io/SublimeWebInspector/)
  • brackets ：安装Theseus 插件支持，支持实时预览，支持记录调用次数和调用栈,包括服务端 JS 的函数调用, 调用的数据和层次都会被记录下来生成树状的图供调试（这是一个亮点）。
  • Webstorm：集成支持，支持添加多对象监视，不支持实时预览。

总结：Webstorm的调试更直接，操作简单，但不支持实时预览。Sublime Text调试复杂，支持实时预览。而brackets 调试简单直观，支持实时预览且可查看函数调用次数和调用栈。

编辑效率

  • Sublime Text ：Zen Coding支持、根据注释生成文档、自定义模板支持、自定义 Snippet 代码、代码格式整理、类似于代码匹配高亮、js css 折叠、粘贴板历史记录、自定义快捷键、多选同时编辑、快速文件切换、快速罗列与定位函数/HTML的元素、集所有功能于一身的命令操作。优点：集所有功能于一身的命令操作。 缺点：对重构代码不支持、不支持函数跳转到定义快捷操作。
  • brackets ：对比Sublime Text无特点突出。缺点：缺点太多，优点不足。
  • Webstorm：sublime的编辑操作都有。 优点：可查看对象数据结构。可查看函数或对象的工程或当前文件的引用。 支持js、css重构插件。支持CSS-X-Fire插件（在firefox中css修改同步保存编辑代码）这个功能很重要，比其它编辑器实时调试更好用。本地代码库可以查看对比最近的编辑内容，但此功能对内存消耗大。
总结：经过对比Sublime Text 与 Webstorm 是两款不错的编辑器也各有特色，Sublime Text轻量级，运行速度比ws快。靠插件来完成很多工作，比较灵活。上手比较容易。对于调试与对目前流行前端框架与测试框架的支持上来说不相上下。但对比ws在前端开发效率来说虽然有集各种功能与一身的类似cmd命令的操作面板的优点，但ws在开发效率上有绝对优势。对于涉及js模块化开发大型项目建议使用webstorm9。对于小型开发或只写点页面效果可以使用sublime Text3.
